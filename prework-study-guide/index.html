HTML

  • HTML means Hypertext Markup Language.
  • !DOCTYPE is a declaration that informs the browser that the file type is HTML. This is not case sensitive, but HTML elements are case sensitive.
  • HTML files usually have an opening and closing tag represented with angle brackets.
  • An element refers to the entire element from opening tag to closing tag.
  • Examples include; text elements, header elements, image elements, list elements
  • A tag refers only to what appears inside the angle brackets.
  • Head elements contain metadata that enables the browser to render the page correctly.
  • Elements inluded with the head element are not visible when a user looks at a webpage.
  • Examples of such elements include; title, style, base, link, script, character set,keywords.

CSS

  • CSS means Cascading Style Sheet
  • Three ways of creating CSS files; Inline, Internal and External CSS.
  • Inline CSS applies to specific HTML tag only.
  • Internal CSS applies to the head section of the HTML file.
  • External CSS allows developers to keep all their CSS rules in a separate file, this makes design changes easier.
  • When using style sheet, we link the file using a (link) element palced in the HTML's (head).
  • CSS Rule: Declaration Block holds declarations which nests the property and their value.
  • Margin indicates how much space around the outside of an element.
  • Padding indicates how much space around the content inside an element.
  • Two or more elements that will share the same styling can be grouped together to help us write "DRY" code.

Git

  • Git command must precede every git statement.
  • git status: checks hat branch we are currently on.
  • git checkout: moves the working branch to a new branch.
  • git -b: creates a new branch.
  • git pull: receives a branch's modifications into the local environment.
  • origin: keyword indicates the source the pull will be in the GitHub repository.
  • git push: creates a new remote branch or add new changes to a existing branch.
  • main: keyword indicates the branch

JavaScript

  • Semi-colon ends JavaScript codes.
  • No semi-colon after a closing curly bracket except i assignment statements.
  • No semi-colon after the round bracket of an IF, FOR, WHILE or SWITCH statements.
  • JS is case sensitive and variables may hold values that have different data types.
  • Data types include; String, Number, Boolean.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Comments; /* before and */ after lines of code makes the inactive and the browser ignores the codes.
  • Operators are mathematical symbols that prodeuces results based on variables.
  • Conditionals are code structure used to test if an expression returns true or not. eg; if...else statement.
  • Functions are a way of packaging functionality tha may be reused, a good alternative to repeatedly writing the same code.
  • Events are code structures that listen for activity in the browser and run code in response. eg; click event, which is fired by the browser when you click something.
  • Array is a single variable that is used hold group of data.
  • The location of the data item in an array is known as array index.
  • Array Index starts from 0 and increases by whole number (1) in ascerding order.
  • For loop must have a starting point, the condition and iteration over each item.
